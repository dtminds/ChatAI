# 快捷话术 Excel 导入设计

## 目标

为快捷话术提供批量导入能力，让运营或客服主管可以通过固定 Excel 模板快速初始化企业话术或个人话术。

首期导入只支持纯文本话术，不支持附件导入。导入能力重点解决：

- 批量创建一级分类和二级分类
- 批量创建纯文本话术
- 自动设置短标题颜色
- 导入前预检，避免错误数据直接入库
- 导入过程中展示前端进度

## 产品边界

首期只支持 `.xlsx` 文件，不支持 `.xls`、CSV、在线表格链接或多 sheet 合并导入。

Excel 模板固定读取第一张 sheet。表头从第一行读取，必须与模板完全一致，不做兼容解析。首期固定要求第一行按顺序只有以下四列：

| 列名 | 单元格是否必填 | 说明 |
| --- | --- | --- |
| 一级分类 | 必填 | 不存在时自动创建一级分类 |
| 二级分类 | 必填 | 不存在时自动创建二级分类 |
| 话术标题 | 选填 | 对应快捷话术的短标题，即 `labelText` |
| 话术内容 | 必填 | 对应快捷话术主文本，即 `contentText` |

表头必须完全等于 `一级分类`、`二级分类`、`话术标题`、`话术内容`。缺列、多列、顺序不一致、列名有额外空格或其它字符，都视为模板错误。

不让客户填写短标题颜色。颜色由系统按导入数据自动分配。

首期不支持：

- 图片、文件、H5、小程序、视频号附件导入
- 解析 Excel 单元格颜色、字体、合并单元格样式
- 多标签
- 跨 sheet 导入
- 导入后自动去重合并已有话术
- 后端异步导入任务
- 服务端保存原始 Excel 文件

单个 Excel 最多导入 1000 条有效话术。超过 1000 条时，前端预检直接拦截，提示用户拆分文件后重试。后端批量创建话术接口同样限制单批最多 100 条，不能只依赖前端限制。

## 解析方式

Excel 在前端解析，不上传原始文件到服务端。

流程：

1. 用户在前端选择 `.xlsx` 文件。
2. 前端使用 `file.arrayBuffer()` 读取本地文件。
3. 前端动态加载 Excel 解析库，例如 `read-excel-file`。
4. 前端解析第一张 sheet，并转换为结构化行数据。
5. 前端展示预检结果。
6. 预检通过后，前端停留在确认页，要求用户二次确认“导入 X 条话术”。
7. 用户点击“开始导入”后，前端进入导入执行页并展示进度条。
8. 前端先提交分类树，确保一级和二级分类存在。
9. 前端拿到二级分类 ID 后，按 100 条一批提交话术。
10. 前端根据批次进度更新进度条。
11. 全部批次完成后刷新快捷话术分类和当前一级分类内容。

后端不接收原始 Excel 文件，不保存临时文件，不做文件清理任务。

前端 Excel 解析库应使用动态 import，避免影响聊天工作台首屏 bundle：

```ts
const { default: readXlsxFile } = await import("read-excel-file");
const rows = await readXlsxFile(file);
```

## 导入入口

快捷话术侧边栏增加导入入口。

建议入口位置：

- 放在快捷话术面板顶部工具区，和“企业 / 个人”切换、搜索框在同一块区域
- 顶部第一行布局建议为：`企业 / 个人` 切换 + 搜索框 + 导入 icon button
- 导入按钮放在顶部工具区右侧，使用弱化的 icon button，不占用分类列表空间
- hover tooltip 文案为“导入话术”
- 和“企业 / 个人”当前 scope 绑定
- 用户当前在企业话术，则导入到企业话术
- 用户当前在个人话术，则导入到个人话术
- 导入是面板级管理动作，不放在一级分类或二级分类的右键菜单中

导入弹窗分四步：

1. 选择文件
2. 预检结果
3. 确认导入
4. 导入结果

## 前端预检

前端解析后生成：

```ts
type QuickReplyImportParsedRow = {
  rowNumber: number;
  primaryCategory: string;
  secondaryCategory: string;
  labelText: string;
  contentText: string;
};
```

前端基础校验：

- 表头必须完全等于模板表头：`一级分类`、`二级分类`、`话术标题`、`话术内容`
- `话术标题` 列必须存在，但单元格可以为空，空值按空字符串处理
- 数据区不能出现整行空白，整行空白视为错误行
- 一级分类不能为空
- 二级分类不能为空
- 话术内容不能为空
- 分类名称不超过 10 字
- 话术标题不超过 10 字
- 话术内容不超过 1000 字
- 空白字符需要 trim

预检弹窗展示：

- 总行数
- 可导入行数
- 错误行数
- 将新建的一级分类数量
- 将新建的二级分类数量
- 将创建的话术数量
- 错误明细：行号、错误原因

如果预检通过，确认页必须明确展示二次确认文案：

- `确认导入 X 条话术？`
- 将创建的一级分类数量
- 将创建的二级分类数量
- 将导入的话术数量

用户点击“开始导入”后才调用后端接口。在用户点击前，不创建分类，也不创建话术。

首期采用严格策略：

- 表头不完全符合模板时，不允许导入
- 数据区出现整行空白时，不允许导入
- 只要存在错误行，不允许导入
- 有效话术超过 1000 条时，不允许导入
- 用户修正 Excel 后重新选择文件
- 不做“跳过错误行继续导入”

## 后端接口

首期不做单个后端 import 接口。导入由前端编排，后端只提供两个可复用能力：

1. 批量确保分类存在
2. 批量创建纯文本话术

### 确保分类存在

```http
POST /api/server/quick-replies/categories/ensure
```

请求体：

```json
{
  "scopeType": 1,
  "categories": [
    {
      "title": "客服最常用话术",
      "children": ["接待开场白", "退换货问题"]
    }
  ]
}
```

字段说明：

- `scopeType`：`1` 企业话术，`2` 个人话术
- `categories[].title`：一级分类名称
- `categories[].children`：该一级分类下的二级分类名称列表

响应体：

```json
{
  "ok": true,
  "categories": [
    {
      "id": "10",
      "title": "客服最常用话术",
      "children": [
        {
          "id": "11",
          "title": "接待开场白"
        },
        {
          "id": "12",
          "title": "退换货问题"
        }
      ]
    }
  ],
  "summary": {
    "createdPrimaryCategoryCount": 1,
    "createdSecondaryCategoryCount": 2
  }
}
```

该接口必须幂等：

- 同名一级分类已存在时复用
- 同一个一级分类下同名二级分类已存在时复用
- 重复调用不会创建重复分类
- 首期不处理并发导入下的重复创建竞争，不新增数据库唯一约束或事务型并发保护

### 批量创建话术

```http
POST /api/server/quick-replies/batch
```

请求体：

```json
{
  "scopeType": 1,
  "items": [
    {
      "rowNumber": 2,
      "categoryId": "11",
      "labelText": "接待开场白",
      "labelColor": "orange",
      "contentText": "亲，您好，非常高兴为您服务，有什么可以为您解答？"
    }
  ]
}
```

字段说明：

- `scopeType`：`1` 企业话术，`2` 个人话术
- `items[].rowNumber`：Excel 原始行号，用于错误提示
- `items[].categoryId`：二级分类 ID
- `items[].labelText`：短标题，可为空
- `items[].labelColor`：短标题颜色，由前端按规则生成
- `items[].contentText`：话术主文本

响应体：

```json
{
  "ok": true,
  "summary": {
    "createdQuickReplyCount": 100
  }
}
```

如果校验失败：

```json
{
  "ok": false,
  "errorMsg": "导入数据有误",
  "errors": [
    {
      "rowNumber": 8,
      "message": "二级分类不能为空"
    }
  ]
}
```

## 后端校验

后端不能信任前端预检结果，两个接口都必须重复校验。

两个导入接口的业务校验失败使用 HTTP 200 返回，并在响应体中返回 `ok: false`、`errorMsg` 和可选的 `errors`。认证失败、权限不足、服务异常等非业务校验错误仍走统一错误处理。

分类 ensure 接口校验：

- 一级分类不能为空，且不超过 10 字
- 二级分类不能为空，且不超过 10 字
- `scopeType` 必须是企业或个人话术
- 企业话术写入 `sub_uid = 0`
- 个人话术写入当前子账号 `sub_uid`
- 单次提交一级分类数量和二级分类总数需要限制，建议一级分类最多 100 个，二级分类总数最多 500 个

批量创建话术接口校验：

- `items` 必须是数组
- 单批最多 100 条
- `categoryId` 必须是当前 scope 下的 active 二级分类
- 话术标题不超过 10 字
- `labelColor` 必须是允许的业务 palette key，或空字符串
- 话术内容不能为空，且不超过 1000 字
- `attachments` 不从导入接口传入，后端固定写空数组
- 话术必须挂到二级分类，不允许挂一级分类

后端也应按行返回错误，错误中带 `rowNumber`。

批量创建话术接口如果存在任意错误，不写入该批数据。由于导入由前端分批编排，前面已经成功的分类和话术不会回滚。

## 权限

导入接口沿用现有快捷话术写权限：

- 必须登录并通过 session 校验
- 能新建快捷话术的子账号可以导入话术
- viewer 等无快捷话术写权限的子账号不能导入
- 企业话术和个人话术的可见性、`uid`、`sub_uid` 规则与现有快捷话术创建接口一致

## 分类创建规则

前端从 Excel rows 中提取分类树，并调用分类 ensure 接口。后端按当前 scope 复用或创建分类。

一级分类规则：

- 在当前 `uid + scopeType + subUid` 范围内按分类名称查找 active 一级分类
- 存在则复用
- 不存在则创建
- 新建一级分类追加到当前一级分类列表末尾

二级分类规则：

- 在对应一级分类下按二级分类名称查找 active 二级分类
- 存在则复用
- 不存在则创建
- 新建二级分类追加到该一级分类下二级分类列表末尾

同一个 Excel 内重复出现的分类应在内存中去重，避免重复查库和重复创建。

## 话术创建规则

每一条有效行创建一条快捷话术。

字段映射：

- `category_id`：对应二级分类 ID
- `content_text`：`话术内容`
- `attachments`：空数组
- `label_text`：`话术标题`，可为空
- `label_color`：前端按规则自动分配，后端只校验为允许的 palette key
- `sort`：追加到对应二级分类的话术列表末尾

首期不做内容去重。Excel 中有多少有效行，就创建多少条话术。

前端按 100 条一批调用批量创建话术接口。批次顺序必须保持 Excel 行顺序，避免导入后排序和用户预期不一致。

## 短标题颜色规则

不让客户填写短标题颜色。

导入时由前端以二级分类为单位分配颜色：

1. 按 Excel 行顺序处理同一个二级分类下的话术。
2. 收集该二级分类内去重后的非空短标题。
3. 按短标题首次出现顺序分配颜色。
4. 同一个二级分类内，相同短标题必须同色。
5. 不同二级分类之间互不影响，重新从 palette 第一个颜色开始分配。
6. 短标题为空时，`label_color` 写空字符串，不展示徽标。
7. 短标题数量超过 palette 数量时循环使用。

示例：

| 二级分类 | 话术标题 | 分配颜色 |
| --- | --- | --- |
| 接待开场白 | 接待开场白 | palette[0] |
| 接待开场白 | 是否有货 | palette[1] |
| 接待开场白 | 什么时候发货 | palette[2] |
| 接待开场白 | 接待开场白 | palette[0] |
| 退换货问题 | 接待开场白 | palette[0] |
| 退换货问题 | 质量问题 | palette[1] |

颜色来源使用快捷话术业务 palette，例如 `quick-reply-title-palette.ts` 中定义的预设 key。不要把这套业务颜色提升到全局 theme token。

## 一致性与失败处理

首期导入不是事务型整体导入，也不做后端异步任务。

导入由前端按步骤编排：

1. 前端解析并预检。
2. 前端展示确认页，等待用户点击“开始导入”。
3. 前端调用分类 ensure 接口。
4. 分类确保成功后，前端按 100 条一批调用批量创建话术接口。
5. 任意批次失败时，导入流程停止。

失败语义：

- 分类创建成功后，如果话术导入失败，分类会保留。
- 前几批话术成功后，如果后续批次失败，已成功的话术会保留。
- 用户刷新页面、关闭页面或断网，导入中断。
- 用户重新导入同一个 Excel，可能重复创建话术。
- 首期不处理重复导入和断点续传。

失败提示：

- 如果分类 ensure 失败，提示“分类创建失败，请重试”
- 如果第 N 批话术失败，提示“导入中断，已成功导入 X 条，请检查后重试”

该策略符合首期边界：导入是批量录入工具，不保证整体事务，不保证断点续传。

## 导入进度

前端导入时展示进度条。

进度条只在用户点击“开始导入”后展示。解析完成和预检确认阶段不展示导入进度条，只展示预检摘要和确认信息。

进度分两段：

- 分类阶段：占 10%
- 话术批量创建阶段：占 90%

示例：

```ts
progress = 10 + Math.floor((createdQuickReplyCount / totalQuickReplyCount) * 90);
```

进度状态：

- `idle`：未选择文件
- `parsed`：解析完成，展示预检结果
- `confirming`：预检通过，等待用户确认导入
- `ensuringCategories`：正在创建分类
- `importingReplies`：正在导入话术
- `success`：导入完成
- `failed`：导入失败

导入过程中：

- 禁用“开始导入”按钮和再次选择文件
- 展示当前进度百分比
- 展示文案，例如“正在导入 300 / 850 条”
- 如果失败，展示已成功导入条数

## 前端展示

导入弹窗的预检页展示结构：

- 顶部摘要：可导入数量、将新建分类数量、错误数量
- 错误列表：行号 + 原因
- 可导入预览：展示前若干行，例如前 20 行

如果存在错误：

- 主按钮禁用
- 文案为“请修正错误后重新上传”

如果无错误：

- 主按钮为“开始导入”
- 点击前展示二次确认文案，例如“确认导入 X 条话术？”
- 导入中展示进度条
- 导入完成后展示导入结果
- 关闭弹窗后刷新快捷话术分类和当前一级分类内容

## 测试要求

前端测试：

- 能解析合法模板并展示预检摘要
- 表头缺列、多列、顺序不一致或列名不完全一致时报错
- 数据区出现整行空白时报错
- 必填列为空时报错
- 超长字段时报错
- 有错误行时不允许提交
- 超过 1000 条有效话术时不允许提交
- 分类 ensure 成功后按 100 条一批提交话术
- 导入过程中展示进度
- 中途某批失败时停止后续批次并展示已成功数量
- 导入成功后刷新快捷话术数据

后端测试：

- 创建缺失一级分类和二级分类
- 复用已有一级分类和二级分类
- 分类 ensure 接口重复调用不创建重复分类
- 分类 ensure 接口业务校验失败返回 `ok: false`
- 话术必须挂二级分类
- 分类 10 字、话术标题 10 字、话术内容 1000 字长度校验
- 批量创建话术单批最多 100 条
- 批量创建话术接口不接受附件
- 批量创建话术接口校验 `labelColor` 为允许的 palette key
- 批量创建话术接口任意错误存在时不写入该批，并返回 `ok: false`

## 后续扩展

后续可以考虑：

- 支持下载标准 Excel 模板
- 支持错误行导出
- 支持“跳过错误行导入”
- 支持重复话术检测
- 支持导入附件，但需要单独设计，不应复用纯文本导入模板
- 支持服务端保存原始 Excel 文件用于审计
- 支持后端异步导入任务和断点续传
- 支持事务型整体导入
