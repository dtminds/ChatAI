# 群聊消息展示差异化设计

- 日期：2026-05-11
- 状态：Draft
- 适用范围：`/chat` 工作台消息展示层
- 目标：修正群聊消息左右布局、头像和昵称展示规则，使其与单聊展示逻辑区分开

## 1. 背景

当前 `/chat` 工作台的消息列表对单聊和群聊使用同一套渲染逻辑，只根据 `message.role` 决定左右布局和头像位置。

这会导致群聊页面出现两个问题：

1. 当前席位发送的消息没有稳定落在右侧。
2. 群聊消息没有按多人场景展示昵称，导致消息归属不清晰。

单聊与群聊的展示规则不同，不能继续只依赖 `customer / agent / system` 的二分渲染。

## 2. 目标

### 2.1 群聊展示规则

- 当前席位发送的消息显示在右侧。
- 其他成员的消息显示在左侧。
- 左侧群聊消息展示头像和昵称。
- 右侧当前席位消息不展示昵称。

### 2.2 单聊展示规则

- 保持现有左右布局。
- 保持现有头像展示。
- 不展示昵称。

## 3. 方案选择

采用方案 A：

- 在消息适配层补充群聊展示所需的派生字段。
- 在消息行组件中引入会话模式判断。
- 在渲染层统一处理“群聊是否显名”和“是否为当前席位消息”。

不采用的方案：

- 纯前端临时推断：规则分散，后续容易失控。
- 直接改后端契约为全新消息模型：改动面过大，不适合作为这次修复的第一步。

## 4. 数据与渲染设计

### 4.1 适配层新增派生语义

消息适配层需要输出足够的展示信息，至少包括：

- `isGroupConversation`
- `isOwnMessage`
- `displayName`
- `displayAvatarUrl`

其中：

- `isGroupConversation` 由当前会话模式决定。
- `isOwnMessage` 用于区分当前席位消息和其他成员消息。
- `displayName` 仅在群聊左侧消息使用。
- `displayAvatarUrl` 继续沿用消息发送者头像。

### 4.2 消息行组件规则

消息行组件根据以下规则渲染：

- `single`：沿用现有布局。
- `group && isOwnMessage`：右对齐，不展示昵称。
- `group && !isOwnMessage`：左对齐，展示头像和昵称。

系统消息保持居中展示，不参与左右规则。

### 4.3 昵称展示位置

群聊左侧消息的昵称放在头像和气泡之间的辅助信息位置，避免和气泡内容混在一起。

昵称只作为归属信息，不作为消息正文的一部分。

## 5. 影响范围

本次修改只影响消息展示链路：

- `apps/web/src/pages/chat/api/workbench-adapter.ts`
- `apps/web/src/pages/chat/chat-types.ts`
- `apps/web/src/pages/chat/components/message-feed.tsx`
- 必要时补充 `apps/web/src/pages/chat/components/message/*`

不调整：

- 会话列表排序
- 会话摘要
- 后端消息查询条件
- 轮询协议
- 群成员 composer 提及逻辑

## 6. 风险与约束

- 如果 `isOwnMessage` 的判断依据不稳定，会出现群聊左右方向再次错位。
- 如果昵称展示没有单独排版，群聊长昵称可能挤压气泡内容。
- 如果把单聊也强行显名，会破坏现有信息密度和视觉习惯。

因此本次必须严格限定：

- 仅群聊左侧消息显名
- 单聊保持不显名
- 当前席位消息只在右侧展示

## 7. 验证方式

至少覆盖以下场景：

1. 单聊消息列表保持现有外观。
2. 群聊里当前席位发言显示在右侧。
3. 群聊里其他成员发言显示在左侧。
4. 群聊左侧消息显示昵称和头像。
5. 群聊右侧消息不显示昵称。
6. 系统消息仍居中展示。

## 8. 结论

本次修复应以“群聊消息渲染差异化”为边界，优先在适配层补齐展示语义，再在消息行组件中落地群聊专用渲染规则。
